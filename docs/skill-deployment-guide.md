# 试卷解析 Skill 部署架构说明

## 问题与解决方案

### 问题
Claude Code **Skills** 只能在本地 Claude Code 环境中运行，不能直接部署到服务器。但应用的核心解析能力应该可以在两种环境中使用：
1. **本地开发** - 使用 Claude Code Skills 进行交互式开发和调试
2. **服务器部署** - 通过 API 路由提供 HTTP 接口

### 解决方案：双接口架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          试卷解析能力 (Core Logic)                      │
│ ├─ OCR 识别                                                              │ │
│ ├─ 题目提取                                                              │ │
│ ├─ 学科推断                                                              │
│ └─ 数据保存                                                              │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                    ┌───────────────┴───────────────┐
                    │                               │
        ┌───────────▼────────┐          ┌───────▼──────────┐
        │ Claude Code Skill      │          │  Next.js API Route  │
        │ (本地开发环境)         │          │  (服务器部署)       │
        └───────────┬────────┘          └───────┬──────────┘
                    │                               │
                    v                               v
        ┌───────────────────────┐  ┌───────────────────────────┐
        │ - 命令行调用            │  │ - HTTP POST /api/...       │
        │ - 右键菜单             │  │ - 前端 fetch()          │
        │ - 交互式调试           │  │ - 移动端 App            │
        └───────────────────────┘  └───────────────────────────┘
```

## 部署方式

### 方式 1：嵌入式部署（当前推荐）✅

```
┌─────────────────────────────────────────────────────────────┐
│                   单一 Next.js 应用                           │
│                                                               │
│  ┌──────────────┐   ┌──────────────┐   ┌──────────────┐      │
│  │  前端页面      │   │  API 路由    │   │  核心逻辑    │      │
│  │  React/Next.js │ ──→│  /api/exam/   │   │  /skills/     │      │
│  └──────────────┘   │  parse-image  │───│  parse-exam/  │      │
│                      └──────────────┘   └──────────────┘      │
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │  外部依赖                                            │  │
│  │  ├─ 通义千问 API (dashscope)                           │  │
│  │  ├─ 文件系统存储 (data/exams/)                         │  │
│  │  └─ 环境变量配置 (.env)                              │  │
│  └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

**部署步骤：**
```bash
# 1. 构建应用
npm run build

# 2. 启动服务器（或部署到 Vercel/Node.js 服务器）
npm start

# 3. Skill 在本地开发时可用，API 在生产环境可用
```

## 核心文件说明

### lib/skills/parse-exam/

| 文件 | 说明 | 部署 |
|------|------|------|
| `core.ts` | 核心解析逻辑，不依赖任何框架 | ✅ 随应用打包 |
| `api-adapter.ts` | Next.js API 适配器 | ✅ 随应用打包 |
| `skill-adapter.ts` | Claude Code Skill 适配器 | ❌ 仅本地开发 |
| `index.ts` | 模块导出 | ✅ 随应用打包 |
| `skill.json` | Skill 元数据 | ✅ 随应用打包 |

### 关键设计原则

1. **核心逻辑独立**
   - `core.ts` 不依赖 Next.js 或 Claude Code
   - 可以在任何 Node.js 环境中运行
   - 通过依赖注入实现灵活性

2. **适配器模式**
   - API 适配器：处理 HTTP 请求/响应
   - Skill 适配器：处理 Claude Code 交互
   - 两者都调用相同的核心逻辑

3. **配置外部化**
   - AI 提供商通过环境变量配置
   - 支持多种 AI 模型（通义千问、OpenAI 等）

## 本地开发流程

### 使用 Claude Code Skill（推荐）

```bash
# 1. 打开 Claude Code
# 2. 导航到项目目录

# 3. 方式一：命令行调用
/parse-exam --file "./data/test-exam.jpg"

# 4. 方式二：右键菜单
# 在文件管理器中右键点击图片
# 选择 "Claude Code > 解析试卷"
```

### 使用 API 调试（前端开发）

```bash
# 1. 启动开发服务器
npm run dev

# 2. 使用 curl 测试
curl -X POST http://localhost:3000/api/exam/parse-image \\
  -F "file=@./test.jpg" \\
  -F "subject=数学"

# 3. 或通过前端页面上传测试
```

## 环境变量配置

```env
# AI 提供商选择
AI_PROVIDER=dashscope

# 通义千问配置
DASHSCOPE_API_KEY=your-api-key
DASHSCOPE_MODEL=qwen3.5-plus
```

## 总结

✅ **可行性确认**：封装成 skill 后，可以与应用一起部署

- 核心解析逻辑在 `core.ts` 中，与 Claude Code 解耦
- 通过 API 适配器提供 HTTP 接口，支持服务器部署
- Skill 适配器仅在本地开发时使用，不参与生产部署
- 整个解析能力与应用一起打包，无需额外配置

**部署时**：
- 只需部署 Next.js 应用
- API 路由自动使用核心解析逻辑
- Skill 功能保留用于本地开发调试
